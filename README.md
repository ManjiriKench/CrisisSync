# CrisisSync: Real-Time Emergency Response Platform

*A streamlined, AI-driven incident tracking and management system built for rapid emergency response.*

---

## 🎯 The Problem (Slide 1)
During emergencies, response times are often delayed due to fragmented communication, manual triage, and lack of real-time situational awareness. Staff and responders need a unified system to track incidents, coordinate resources, and react instantly.

## 💡 The Solution (Slide 2)
**CrisisSync** is a full-stack emergency management platform that bridges the gap between individuals in distress and response teams. It provides a seamless flow of information from SOS reports to real-time dashboard analytics, allowing for automated triage and swift resolution.

---

## 🚀 Key Features (Slide 3 & 4)

### 1. Unified Command Dashboard (Web)
* **Live Incident Map:** Real-time geolocation tracking of active incidents.
* **Live Feed & Triage:** Automated categorization and prioritization of emergencies (AI-simulated triage).
* **Staff Management:** Assign, track, and monitor response personnel efficiently.

### 2. Dual-Purpose Mobile App
* **Guest SOS Mode:** Quick, intuitive incident reporting for people in distress with instant location sharing.
* **Staff Responder Mode:** Real-time alerts, incident details, and resolution tracking for emergency responders on the ground.

---

## 🛠️ Tech Stack (Slide 5)
* **Frontend (Command Center):** React.js, Vite, Zustand (State Management), Recharts (Data Viz)
* **Frontend (Mobile):** React Native, Expo
* **Backend:** Node.js, Express.js
* **Architecture Design:** Built with a demo-ready mock-first architecture to reliably simulate real-time tracking, AI triage, and automated escalation flows.

---

## 📈 Impact & Value Proposition (Slide 6)
* **Drastically Reduced Response Time:** Automated triage removes manual bottlenecks.
* **Total Situational Awareness:** Live mapping gives command centers a bird's-eye view of all crises.
* **Highly Scalable:** Designed to easily plug into real-world AI processing and external emergency service APIs in the future.
